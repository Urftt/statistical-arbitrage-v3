## Spacing Scale

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding |
| sm | 8px | Slider label-to-value gap, compact row padding |
| md | 16px | Default Stack gap between form elements and between cards |
| lg | 24px | Section gap between parameter group labels and between results blocks |
| xl | 32px | Gap between the parameter form and results area |

| 3xl | 64px | Not used in this phase |

Exceptions: Mantine `Slider` thumb touch target is rendered at 20px diameter by the library — do not override. Chart margin follows `PLOTLY_DARK_TEMPLATE` defaults: `{ t: 48, b: 40, l: 56, r: 24 }`.

## Typography

Source: `frontend/src/lib/theme.ts` (Mantine dark theme); consistent with Phase 2.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 (regular) | 1.5 |
| Label | 12px | 400 (regular) | 1.4 |
| Heading (section) | 16px | 600 (semibold) | 1.2 |
| Display (stat value) | 20px | 600 (semibold) | 1.2 |

Notes:
- "Body" covers slider labels, table cell text, alert body copy.
- "Label" covers slider current-value readout, badge text, table column headers.
- "Heading" covers parameter group labels ("Signal", "Risk Management", "Execution") and results section titles.
- "Display" covers the numeric value inside metric cards (Sharpe, P&L, etc.).
- Mantine handles font-size via component props (`size="sm"`, `size="md"`) — map `sm` → 14px, `xs` → 12px, `md` → 16px.

## Interaction Contracts

### Parameter Form

- All 7 sliders update their current-value readout in real time as the thumb moves.
- "Run Backtest" triggers `postBacktest()` immediately on click with no debounce.
- "Run Backtest" is disabled and shows a `Loader` size="xs" inline while a backtest is in flight.
- "Reset to Defaults" restores all slider state to `DEFAULT_STRATEGY_PARAMETERS` values and clears any existing results (results area returns to empty state).
- Changing the pair in `PairContext` clears results and resets sliders to defaults (consistent with Phase 2 pair-change behavior per NAV-05).

### Warning Display Order

1. Preflight blocker alerts appear immediately below the button row. If any blocker is present, stop here — no results render.
2. Preflight warning alerts appear immediately below the button row (above results) if status is `'ok'` but warnings exist.
3. Results render in order: context label → metric cards → overfitting warning banners (if any) → equity curve → drawdown → z-score → spread → trade log → Assumptions accordion.

### Assumptions Accordion

- Collapsed by default.
- On expand: renders `footer.execution_model`, `footer.fee_model`, `footer.data_basis` as labeled `Text` rows, then `footer.assumptions[]` and `footer.limitations[]` as bulleted lists.
