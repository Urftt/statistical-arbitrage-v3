## Spacing Scale

Mantine v8 spacing tokens (8-point system). Declared values used in this phase:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, inline Badge padding |
| sm | 8px | Stack gap inside metric cards, form field internal padding |
| md | 16px | `Paper p="md"` card internal padding, default Stack gap between controls |
| lg | 24px | Stack gap between major sections (axis config → buttons → results) |
| xl | 32px | `Paper p="xl"` empty-state centered content |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Fold table rows use Mantine `Table` default row height (no override needed).
- NumberInput min/max/step fields inside axis configuration panels: `Stack gap="sm"` (8px) between label and inputs, `SimpleGrid cols={3}` with `gap="sm"` for the three range fields.
- `Group gap="md"` (16px) between the two Run buttons.

## Color

All values from `frontend/src/lib/theme.ts` and Mantine dark palette. Dark-only — no light-mode variants needed.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `rgba(26, 27, 30, 1)` — Mantine `dark[7]` | Page background, `AppShell.Main`, Plotly `plot_bgcolor` |
| Secondary (30%) | Mantine `dark[6]` (~`#25262b`) | `Paper withBorder` cards (axis config panel, best cell card, fold table container), Accordion background |
| Accent (10%) | `#339AF0` — Mantine `blue[5]` | Run buttons (primary action), "Apply to Backtest" button, heatmap color scale high end |
| Destructive | `#FF6B6B` — Mantine `red[5]` | Fragile verdict badge, blocker Alert banners, heatmap color scale low end (negative Sharpe) |

Accent reserved for:
1. "Run Grid Search" button (`variant="filled"`)
2. "Run Walk-Forward" button (`variant="filled"`)
3. "Apply to Backtest" button on best cell card (`variant="light"` to distinguish from primary Run actions)
4. "Apply to Backtest" button on walk-forward results (`variant="light"`, only visible when verdict is stable/moderate)
5. Heatmap colorscale warm end (high Sharpe / high P&L)

Semantic color mapping for stability verdict badges:
- Stable: `color="green"` (Mantine `green[5]` = `#51CF66`)
- Moderate: `color="yellow"` (Mantine `yellow[5]` = `#FCC419`)
- Fragile: `color="red"` (Mantine `red[5]` = `#FF6B6B`)

Robustness score badge thresholds (shown on best cell card):
- Strong: robustness_score >= 0.7 → `color="green"`
- Moderate: robustness_score >= 0.4 → `color="yellow"`
- Weak: robustness_score < 0.4 → `color="red"`

Walk-forward fold row status badge:
- `ok` → `color="green"` label "OK"
- `no_train_trades` or `no_test_trades` → `color="yellow"` label "No Trades"
- `blocked` → `color="red"` label "Blocked"

## Component Inventory

All components sourced from existing codebase patterns. No new dependencies.

### Axis Configuration Section
- `Paper withBorder p="md"` wrapper for the entire axis config panel
- Two columns via `SimpleGrid cols={2} spacing="lg"`: each column holds one axis config
- Per-axis column: `Select` for parameter name + `SimpleGrid cols={3} spacing="sm"` for min/max/step `NumberInput` fields
- Labels: "Parameter 1" / "Parameter 2" as `Text size="sm" fw={600}`
- Sweepable parameter options for `Select`: `entry_threshold`, `exit_threshold`, `lookback_window`, `stop_loss`, `position_size`

Default min/max/step values per parameter (Claude's discretion, applied as default values):

| Parameter | Min | Max | Step | Unit |
|-----------|-----|-----|------|------|
| entry_threshold | 1.0 | 3.0 | 0.5 | — |
| exit_threshold | 0.1 | 1.0 | 0.1 | — |
| lookback_window | 20 | 120 | 10 | bars |
| stop_loss | 2.0 | 5.0 | 0.5 | — |
| position_size | 0.2 | 0.8 | 0.1 | — |

### Run Buttons
- `Group gap="md"` containing two `Button` components
- "Run Grid Search": `variant="filled" color="blue"` with `IconPlayerPlay` left icon, `loading` prop during fetch
- "Run Walk-Forward": `variant="filled" color="blue"` with `IconPlayerPlay` left icon, `loading` prop during fetch (independent loading state from grid search)

### Metric Selector (above heatmap)
- `Select` with label "Color by" and options: Sharpe Ratio, Total P&L, Win Rate, Max Drawdown
- Positioned in a `Group justify="space-between"` row alongside the section heading
- Re-colors heatmap client-side from already-loaded `GridSearchResponse.cells` data

### Best Cell Card
- `Paper withBorder p="md"` with `bg="dark.6"` tint
- `Group justify="space-between"` top row: "Best Parameters (In-Sample)" heading + robustness `Badge`
- `SimpleGrid cols={4} spacing="sm"` showing: best param 1 value, best param 2 value, Sharpe, Total P&L
- Each metric cell: `Text size="xs" c="dimmed"` label above `Text size="md" fw={600}` value
- "Apply to Backtest" `Button variant="light" color="blue" size="sm"` right-aligned below metrics

### Heatmap Chart
- Height: 360px
- Plotly `heatmap` trace type
- Colorscale: `RdYlGn` (red = low, green = high) for Sharpe and P&L; `Blues` for Win Rate; reversed `RdYlGn` for Max Drawdown (lower drawdown = better)
- Star annotation on best cell: Plotly `annotations` array with `text: "★"`, `font.size: 18`, `font.color: "#FCC419"`, `showarrow: false`
- X-axis: Parameter 2 values, Y-axis: Parameter 1 values
- Hover template shows: param1 value, param2 value, Sharpe, P&L, win rate, trade count
- Cells with `status !== "ok"` rendered as null (no color) to avoid misleading values
- Wrapped in `PlotlyChart` with `style={{ height: 360 }}`

### Warning Alert Banners
- Mantine `Alert` with `variant="light"` between Run buttons and results sections
- Blocker warnings: `color="red"` with `IconAlertTriangle`
- Non-blocker warnings: `color="yellow"` with `IconAlertTriangle`
- Grid search warnings placed between Run buttons and best cell card
- Walk-forward warnings placed between walk-forward controls and fold table

### Walk-Forward Controls
- `Group gap="md"` inline above fold table
- Fold count: `NumberInput` with label "Folds", min=2, max=10, step=1, default=5, width ~80px
- Train %: `Slider` with label "Train %", min=50, max=80, step=5, default=60, marks at 50/60/70/80, width ~200px
- Both controls disabled while walk-forward is loading

### Walk-Forward Fold Table
- Mantine `Table striped highlightOnHover` inside `Paper withBorder p="md"` wrapper
- Columns: Fold, Train Sharpe, Test Sharpe, Train Trades, Test Trades, Status
- Column widths: Fold (60px), Train/Test Sharpe (100px each), Train/Test Trades (80px each), Status (100px)
- Status column uses `Badge` component per the semantic color mapping above
- Sharpe values formatted to 2 decimal places; trade counts as integers
- Below table: `Group gap="xl"` showing aggregate train Sharpe, aggregate test Sharpe, stability verdict `Badge`
- Stability verdict `Badge size="lg"` with appropriate color (green/yellow/red)

### Honest Reporting Accordion
- Two separate Accordion sections, one per result type (grid search, walk-forward), matching Phase 3 pattern
- `Accordion variant="separated"` collapsed by default
- Panel label: "Assumptions & Limitations"
- Content: render `footer.notes` from API response as `List` items with `Text size="sm" c="dimmed"`

### Loading Skeleton
- Grid search loading: `Skeleton height={360}` (matches heatmap height) below buttons while `gridLoading === true`
- Walk-forward loading: `Skeleton height={200}` (matches fold table approximate height) while `wfLoading === true`
- Both skeletons shown only while respective loading flags are true

### Empty State (no results yet)
- `Paper p="xl" ta="center"` card matching Phase 3 empty state pattern
- Shown when neither grid search nor walk-forward have results yet
- Heading and body copy defined in Copywriting Contract below

## Walk-Forward Index-to-Timestamp Strategy

The API returns `train_start_idx`, `train_end_idx`, `test_start_idx`, `test_end_idx` as bar indices, not timestamps (STATE.md blocker). Display strategy for the fold table:

- Do NOT show raw bar indices as dates — they are unreadable.
- Show fold index only (1, 2, 3…) and Sharpe/trade count columns.
- Omit start/end columns entirely from the table until a timestamp mapping solution is available.
- This is an intentional deferral. The fold table remains useful without date ranges because stability verdict, Sharpe divergence, and trade count per fold are the decision-relevant outputs.

## Interaction Contracts

| Interaction | Behaviour |
|-------------|-----------|
| Run Grid Search click | Disable both Run buttons, set `gridLoading=true`, show Skeleton, call `postGridSearch()`, on success render results, on error show Alert with error copy, re-enable buttons |
| Run Walk-Forward click | Disable both Run buttons, set `wfLoading=true`, show Skeleton, call `postWalkForward()`, on success render results, on error show Alert with error copy, re-enable buttons |
| Metric selector change | Re-map `cells` to new metric client-side, re-render heatmap trace z-values, no API call |
| Parameter 1 / 2 Select change | Update local axis state, reset grid search and walk-forward results (clear stale data), do not auto-run |
| Apply to Backtest (best cell) | Call `onApplyToBacktest(gridData.recommended_backtest_params)`, switch to Backtest tab |
| Apply to Backtest (walk-forward) | Call `onApplyToBacktest(wfData.recommended_params)`, switch to Backtest tab — button hidden when verdict is "fragile" |
| Pair change (via PairContext) | Clear all OptimizeTab state (gridData, wfData, errors, loading) — same pattern as BacktestTab pair-change useEffect |
| Heatmap hover | Plotly native hover tooltip — no custom handler needed |
