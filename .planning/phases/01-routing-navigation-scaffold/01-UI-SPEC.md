## Spacing Scale

Mantine v8 spacing tokens map directly to the 8-point scale. Use Mantine prop values (`xs`, `sm`, `md`, `lg`, `xl`) — never raw pixel values in component props.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, tab icon gaps |
| sm | 8px | Stack gap between compact elements, NavLink internal gaps |
| md | 16px | Default Stack gap, Container padding (`py="md"`) |
| lg | 24px | Section separation within the Pair Analysis page |
| xl | 32px | Major layout gaps |
| 2xl | 48px | Not used in Phase 1 scaffold |
| 3xl | 64px | Not used in Phase 1 scaffold |

Exceptions:
- AppShell header height: 60px (locked in `layout.tsx`, not a spacing token)
- AppShell sidebar width: 260px (locked in `layout.tsx`, not a spacing token)
- NavLink section padding: `p="sm"` on the Stack wrapper (existing pattern from Sidebar.tsx)
- Tab icon size: 16px (matches Tabler icon `size={16}` pattern used in Header.tsx)

Source: `frontend/src/app/(dashboard)/layout.tsx`, `frontend/src/components/layout/Sidebar.tsx`

## Typography

Dark mode only. All values from Mantine theme defaults + observed usage in existing components.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (`size="sm"`) | 400 (regular) | 1.5 |
| Label | 12px (`size="xs"`) | 700 (bold) — for section headers (uppercase labels) | 1.4 |
| Heading | 20px (`size="lg"` or `h3`) | 700 (semibold/bold) | 1.2 |
| Display | 24px (`h2`) | 700 (bold) — Pair Analysis page title "ETH / BTC" | 1.2 |

Usage in this phase:
- Sidebar section labels (`Text size="xs" fw={700} c="dimmed" tt="uppercase"`): 12px, weight 700, uppercase — existing pattern, do not change
- NavLink labels: 14px, weight 400 — Mantine default
- NavLink descriptions: 12px, weight 400, `c="dimmed"` — Mantine default
- Pair Analysis page title (e.g. "ETH / BTC"): 24px `h2`, weight 700
- Tab labels: 14px (`size="sm"`), weight 500 (Mantine pills tab default)
- Placeholder text in tab panels: 14px, `c="dimmed"`, weight 400

Source: `frontend/src/components/layout/Sidebar.tsx` (existing patterns), `frontend/src/lib/theme.ts`

## Interaction Contracts

### Scanner Table Row
- `cursor: pointer` on hover (Mantine prop: `style={{ cursor: 'pointer' }}` on `Table.Tr`)
- Full row is the click target — no separate button
- `onClick` navigates to `/pair-analysis?asset1={base1}&asset2={base2}&timeframe={timeframe}` via `router.push`
- No visual affordance change on hover beyond cursor (existing table styling sufficient)

Source: CONTEXT.md D-03, D-04

### Sidebar Active State
- Active item: Mantine NavLink `active={true}` + `variant="light"` renders accent background
- Scanner: `active={pathname === '/scanner'}`
- Pair Analysis: `active={pathname.startsWith('/pair-analysis')}` — stays highlighted across all tab states and URL params

Source: CONTEXT.md D-01, D-02; RESEARCH.md Pattern 3

### Tab Switching
- `variant="pills"` — pill-shaped tab labels, selected tab gets accent background
- Active tab stored in URL: `?tab=statistics` (default when absent)
- Tab change uses `router.replace` (not `router.push`) — does not pollute browser history
- Pair change via `key={`${asset1}-${asset2}`}` on `<Tabs>` container triggers full remount — all tab state cleared

Source: CONTEXT.md D-05, D-06, D-07, D-09

### Pair Analysis Page Header
Decision (Claude's discretion from CONTEXT.md): Show a `Title` component at the top of the Pair Analysis content area displaying the currently selected pair. Format: "ETH / BTC" (asset1 + " / " + asset2). Use `order={2}` (24px). When no pair is selected (both empty), render `c="dimmed"` text: "Select a pair using the header dropdowns to begin analysis."
