## Spacing Scale

Mantine v8 uses its own spacing tokens. The project maps to this 8-point scale:

| Token | Value | Mantine prop | Usage |
|-------|-------|-------------|-------|
| xs | 4px | `gap="xs"` | Icon gaps, badge-to-text gap |
| sm | 8px | `gap="sm"` | Compact element spacing, `mb={4}` slider label gap |
| md | 16px | `gap="md"` | Default element spacing within Paper, Stack gap between form groups |
| lg | 24px | `gap="lg"` | Tab-level Stack gap (primary vertical rhythm) |
| xl | 32px | `gap="xl"` | Major section separation |
| 2xl | 48px | `p="xl"` on Paper | Empty state padding |
| 3xl | 64px | — | Not used in this phase |

Exceptions:
- Accordion items: Mantine default internal padding (no override needed).
- Section group headers: `mb="xs"` (4px) between label and first accordion item in a group.
- Run button in accordion header: `ml="auto"` right-aligned, no explicit gap.

## Plotly Chart Contract

All charts go through `PlotlyChart` wrapper. The `PLOTLY_DARK_TEMPLATE` is auto-merged.

### Chart heights by module

| Module | Chart type | Height |
|--------|-----------|--------|
| Rolling Stability | Line chart (p-value over time) | 260px |
| OOS Validation | Grouped bar chart (formation vs trading p-value per split) | 240px |
| Cointegration Method | Bar chart (test stat per method) | 220px |
| Lookback Window Sweep | Bar chart (crossings or zscore_std per window) | 240px |
| Z-Score Threshold | Heatmap (entry vs exit threshold, metric as color) | 280px |
| Transaction Cost | Bar chart (Sharpe or P&L per fee level) | 220px |
| Spread Method | Bar chart (metric per method) | 220px |
| Timeframe Comparison | Bar chart (metric per timeframe) | 220px |

Chart config defaults (from PlotlyChart wrapper):
- `responsive: true`
- `displayModeBar: false`
- Margin: `t: 48, b: 40, l: 56, r: 24`

### Rolling Stability chart specifics

- Main trace: scatter/lines, `color: '#339AF0'`, p-value on y-axis over time on x-axis
- Reference line: horizontal shape at `y=0.05`, `color: '#339AF0'`, `dash: 'dash'`, labeled "p=0.05 significance"
- Y-axis: `title: { text: 'p-value' }`, range clamped at `[0, 1]`
- Cointegrated windows highlighted: scatter markers with `color: '#51CF66'` where `is_cointegrated === true`

### Skeleton loading state

Each module in loading state shows one `Skeleton` at the chart height defined above. No text skeleton — only one placeholder per module. Label "Running module..." as `Text size="sm" c="dimmed" ta="center"` beneath skeleton.

## Apply to Backtest: Null Params Handling

When `recommended_backtest_params` is `null` for a module:
- Hide the "Apply to Backtest" button entirely (do not render a disabled button)
- The takeaway callout still renders normally

Source: CONTEXT.md Claude's Discretion area. Hiding is cleaner than a disabled button with no tooltip explanation.

## Recommended Badge

When a module's chart has a single best result (e.g., best lookback window bar, best timeframe bar):
- Render a `Badge size="sm" variant="dot" color="blue"` next to the chart title (`Text size="sm" fw={600}`) showing the recommended value
- Format: `Recommended: {value}` (e.g., "Recommended: 90 bars", "Recommended: 4h")
- Only shown after results load. Hidden during loading and empty states.

## Interaction Contract

### Module run button

- Button state: enabled when no module-level loading is active.
- On click: set module loading state, call API, auto-expand accordion panel when results arrive (set controlled `value` on Accordion to include this module's key).
- On error: show `Alert color="red"` inside the accordion panel. Button returns to enabled state.
- On success: render chart + takeaway + Apply to Backtest button. Button returns to enabled state (re-runnable).

### Pair change (NAV-05)

When `asset1`, `asset2`, or `timeframe` changes in `PairContext`:
- Clear all 8 module result states to `null`
- Clear all 8 module error states to `null`
- Clear all 8 module loading states to `false`
- Accordion expansion state: reset to all collapsed (no stale expanded panels showing blank content)

### Apply to Backtest (D-06, D-07, D-08)

- Button click: invoke `onApplyToBacktest(recommended_backtest_params)` callback received as prop
- Page-level handler: set `pendingBacktestParams` state and switch active tab to "Backtest"
- Backtest tab: on mount and on `pendingParams` prop change, if non-null, merge into form state and call `onParamsConsumed()`
- Does NOT auto-run the backtest — only pre-fills the form

### Simultaneous module expansion

- Accordion uses `multiple` prop — any number of panels can be expanded simultaneously
- Each module manages its own independent loading/error/data state via separate `useState` calls
- No cross-module side effects
