# Phase 260407-f10 Plan 01: Fix lookback_window Grid Search Bug Summary

**One-liner:** Replace `model_copy(update=...)` with `StrategyParameters.model_validate(...)` in grid search and walk-forward so Pydantic v2 re-runs int coercion for `lookback_window` float axis values.

## What Was Fixed

**Root cause:** In Pydantic v2, `model_copy(update=...)` bypasses field validators and assigns values directly to model fields without type coercion. `_build_axis_values()` returns `list[float]` for all axes (including `lookback_window`). So a value like `20.0` was assigned directly to the `int`-typed `lookback_window` field. When `build_rolling_strategy_data()` then called `range(lookback_window - 1, n)`, Python raised `TypeError: 'float' object cannot be interpreted as an integer`.

**Why it surfaced as "all cells blocked":** The `except Exception` block in `run_grid_search` catches the `TypeError` and marks the cell `status="blocked"`. With every cell blocked, `best_cell` is `None` and the heatmap renders empty.

## Exact Lines Changed

### optimization.py (line ~108)

**Before:**
```python
cell_params = base_params.model_copy(update=overrides)
```

**After:**
```python
cell_params = StrategyParameters.model_validate(
    {**base_params.model_dump(), **overrides}
)
```

### walkforward.py (line ~223)

**Before:**
```python
test_params = base_params.model_copy(update=best_params)
```

**After:**
```python
test_params = StrategyParameters.model_validate(
    {**base_params.model_dump(), **best_params}
)
```

`model_validate` runs the full Pydantic v2 validator chain. The lax-mode int validator coerces whole-number floats (`20.0` → `20`) — this is the documented safe path for programmatic model construction from mixed-type dicts.

## Deviations from Plan

### Auto-noted: Walk-forward regression test passes on buggy code

**Found during:** Task 1 (RED confirmation)

**Issue:** The walk-forward test cannot fail on the current buggy code at the same time as the optimization tests. The walk-forward bug (`model_copy` in `test_params` assignment) only manifests when `run_grid_search` returns a non-empty `best_params` (i.e., after the grid search fix). On the unpatched code, the train grid search blocks all cells, `best_params = {}`, and `model_copy(update={})` is a no-op — so no TypeError occurs.

**Resolution:** The test was redesigned to assert `status != "blocked"` for each fold, which is a meaningful regression guard after both fixes are in place. The two optimization tests are the primary regression guards for the pre-fix bug state.
